1. Purpose

This evidence artefact records validation of the CDAS Operations Centre introduced in Phase 3Z-F.

The Operations Centre is intended to provide a single-pane, read-only operational view of the Controlled Document Access System, including:

- core CDAS metrics
- email delivery state
- document access workflow backlog
- licence activity
- download link state
- evidence integrity checks
- recent failed emails
- recent pending requests
- recent licences
- recent downloads
- raw operations payload inspection

This validation confirms that the Operations Centre backend endpoint and deployed admin page are present, authenticated, schema-safe, and operationally useful.

2. Implemented Components

Backend:

- worker/src/cdas/operations.js
- GET /api/admin/cdas/operations
- CDAS admin route wiring
- CDAS barrel export wiring
- router recognition for /api/admin/cdas/operations

Frontend:

- src/pages/admin/cdas-operations.astro
- Admin dashboard card in src/pages/admin/index.astro
- Browser-side authenticated fetch using saved admin token
- Admin styling aligned with the existing admin hub style
- Read-only operational dashboard
- Raw payload inspection panel

5. Validation Assessment

Result: PASS WITH NOTICES

The Phase 3Z-F Operations Centre is functioning as intended for its first read-only slice.

Confirmed:

- Backend endpoint is live.
- Endpoint requires admin authentication.
- Endpoint returns ok=true.
- Endpoint returns no query failures.
- Endpoint uses the verified cdas_email_events schema.
- Endpoint reports operational notices rather than false faults.
- No evidence-integrity faults are currently reported.
- Frontend page is deployed.

- Frontend page uses the updated admin-card visual style.
- The Operations Centre is read-only and non-destructive.

6. Known Notices

6.1 pending_verification_backlog

There are pending document access requests that have not completed email verification.

This is an operational backlog notice, not a system fault.

6.2 historical_failed_email_events_present

There is at least one historical failed email event.

The observed failed event was resolved by successful manual retry. Because unresolved_failed_events is 0 and retryable_events is 0, this is not currently a delivery fault.

7. Known Non-Goals for This Slice

This slice does not implement:

- document upload
- private file upload
- licence correction
- licence supersession
- destructive operations
- revoke from Operations Centre
- regenerate PDF from Operations Centre
- payment integration
- automated remediation
- direct mutation from the Operations Centre

Those are future governed workflows and should be implemented separately with validation evidence.
